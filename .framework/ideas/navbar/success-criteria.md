# Success Criteria: Website Navbar

STATUS:  ACCEPTED

## When this is working, I will observe:

- A user on any non-dashboard page can reach the home page,
  blog, and dashboard without using the browser's back button
  or typing a URL
- The navigation is visible and usable on a mobile screen
  without horizontal scrolling or overlapping content
- Navigating between pages does not require a full page reload
  for links within the site

## It is NOT a success if:

- Any non-dashboard page has no visible navigation
- The navigation is absent or broken on screens narrower
  than 375px
- Adding the navigation measurably increases page load time
  on a standard connection

## Notes

Formal user testing is out of scope for this feature.
Acceptance will be visual review across desktop and mobile
viewports.
