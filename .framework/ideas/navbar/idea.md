# Idea: Website Navbar

SLUG:    navbar
STATUS:  ACCEPTED
CREATED: 2026-02-28

## Problem

Visitors to the website land on a page with no way to navigate
to other sections. There is no visible path from the home page
to the blog, or from the blog back to home. Users who want to
reach the dashboard have to know the URL directly.

## Why this matters

First-time visitors have no way to discover what the site offers.
Users who find a blog post via search have no path to the rest of
the site. This limits the site's ability to convert visitors into
engaged users.

## Who is affected

All non-authenticated visitors. Blog readers arriving from
search. Existing users navigating between the blog and dashboard.

## Constraints and context

- Must not affect the dashboard (has its own navigation)
- Current pages: home (`/`), blog (`/blog`), dashboard (`/dashboard`)
- Mobile support required (site is mobile-first)
